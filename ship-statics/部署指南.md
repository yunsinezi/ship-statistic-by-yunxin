## 🌐 云端部署（推荐分享）

### 方案 1: Railway（最简单）

**步骤 1**: 准备 GitHub 仓库
```bash
git init
git add .
git commit -m "Initial commit"
git remote add origin https://github.com/你的用户名/ship-statics.git
git push -u origin master
```

**步骤 2**: 部署到 Railway
1. 访问 https://railway.app
2. 点击 "Start a New Project"
3. 选择 "Deploy from GitHub repo"
4. 选择你的仓库
5. 点击 "Deploy Now"

**步骤 3**: 获取访问链接
- Railway 会自动提供一个域名
- 例如: `https://ship-statics-production.up.railway.app`

**完成！** 现在可以分享链接给同学使用。

### 方案 2: Render（免费）

**步骤 1**: 推送代码到 GitHub（同上）

**步骤 2**: 在 Render 创建服务
1. 访问 https://render.com
2. 点击 "New +" → "Web Service"
3. 连接 GitHub 仓库
4. 配置：
   - **Name**: ship-statics
   - **Build Command**: `pip install -r requirements.txt`
   - **Start Command**: `python main.py`
   - **Environment**: Python 3

**步骤 3**: 点击 "Create Web Service"

**步骤 4**: 等待部署完成，获取访问链接

## 🐛 常见问题

### Q1: 本地启动失败

**检查**:
- Python 是否安装（运行 `python --version`）
- 依赖是否安装（运行 `pip install -r requirements.txt`）
- 端口 5000 是否被占用

**解决**:
```bash
# 重新安装依赖
pip install -r requirements.txt -i https://pypi.tuna.tsinghua.edu.cn/simple

# 手动启动
python main.py
```

### Q2: 文件上传失败

**检查**:
- 文件是否为 .xlsx 格式
- 文件大小是否超过 50MB
- 后端是否正常启动

**解决**:
- 确认使用模板生成的 Excel
- 检查 Excel 格式是否正确
- 查看浏览器控制台错误信息

### Q3: 云端部署失败

**检查**:
- GitHub 仓库是否公开
- requirements.txt 是否完整
- Procfile 是否正确

**解决**:
- 确保 `app.py` 存在
- 确保 `main.py` 存在
- 查看 Railway/Render 构建日志

### Q4: 云端访问 404

**原因**:
- 部署未成功
- 路由配置错误

**解决**:
- 检查部署日志
- 确认 `main.py` 监听 `0.0.0.0`
- 确认 PORT 环境变量正确
